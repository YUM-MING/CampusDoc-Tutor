# ETC.md

## 1. 프로젝트 진행하면서 느낀 점 (Reflections)
*   **RAG의 강력함**: 별도의 학습 없이도 내부 문서를 참조하여 정확한 답변을 내놓는 RAG 시스템의 효율성을 체감했습니다.
*   **오픈소스 LLM의 발전**: OpenAI 등 상용 API 없이도, Ollama와 Llama 3를 활용해 로컬에서 준수한 성능의 AI 서비스를 구축할 수 있었습니다.
*   **UX의 중요성**: 기능 구현만큼이나, 파일 업로드 시의 피드백, 직관적인 버튼 배치, 추천 질문 등 사용자 경험(UX) 요소가 서비스 완성도에 큰 영향을 미침을 알게 되었습니다.

## 2. AntiGravity 장단점 (AntiGravity Pros & Cons)

### 장점 (Pros)
*   **빠른 구현 속도**: 복잡한 설정이나 보일러플레이트 코드를 순식간에 작성하고, 에러 발생 시 즉각적으로 해결책을 제시하여 개발 시간을 획기적으로 단축시켰습니다.
*   **문맥 파악 능력**: 프로젝트의 이전 맥락을 기억하고, 사용자의 의도(예: 한국어화, 파일 검증 기능 등)를 정확히 파악하여 선제적으로 제안하고 수정했습니다.
*   **유연한 대처**: 단순 코딩뿐만 아니라, 폴더 구조 정리, 문서화(MD 파일 작성), UI 디자인 개선 등 전방위적인 업무 지원이 가능했습니다.

### 단점 (Cons)
*   **디테일 누락**: 빠른 수정 과정에서 필요한 import 문을 빠뜨리거나(예: `ChatPromptTemplate` 누락), CSS 괄호를 닫지 않는 등 꼼꼼한 확인이 필요한 실수가 간혹 발생했습니다. (물론 지적 시 바로 수정됨)
*   **초기 설정의 복잡성**: 사용자의 로컬 환경(파일 경로, 라이브러리 버전 등)을 완벽히 알 수 없으므로, 초기 환경 설정 시 몇 번의 시행착오(경로 문제, 패키지 설치 등)가 필요했습니다.
